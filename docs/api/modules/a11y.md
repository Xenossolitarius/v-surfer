# A11y

Accessibility wiring: ARIA roles, labels, and a screen-reader live region. Module object `A11yModule`, component `<A11y>`, config key `a11y`. It renders no visible chrome — it mutates ARIA attributes and injects a visually-hidden live region.

## Usage

```vue
<script setup>
import { Surfer, Item, A11y, A11yModule } from 'v-surfer'
import 'v-surfer/css'
import 'v-surfer/css/a11y'
</script>

<template>
  <Surfer :modules="[A11yModule]" :slides-per-view="1">
    <Item v-for="n in 5" :key="n">Slide {{ n }}</Item>
    <A11y />
  </Surfer>
</template>
```

## Config parameters

Passed as `<A11y>` props or via `:config="{ a11y: { … } }"`. Message strings support `{{index}}` / `{{slidesLength}}` placeholders.

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `enabled` | `boolean` | `true` | Activate a11y wiring |
| `slideRole` | `string \| null` | `'group'` | `role` for each slide |
| `containerRole` | `string \| null` | `null` | `role` for the container |
| `containerMessage` | `string \| null` | `null` | `aria-label` for the container |
| `containerRoleDescriptionMessage` | `string \| null` | `null` | `aria-roledescription` for the container |
| `itemRoleDescriptionMessage` | `string \| null` | `null` | `aria-roledescription` for each slide |
| `slideLabelMessage` | `string` | `'{{index}} / {{slidesLength}}'` | Per-slide label |
| `prevSlideMessage` | `string` | `'Previous slide'` | Prev-button label |
| `nextSlideMessage` | `string` | `'Next slide'` | Next-button label |
| `firstSlideMessage` | `string` | `'This is the first slide'` | Announced at the start |
| `lastSlideMessage` | `string` | `'This is the last slide'` | Announced at the end |
| `paginationBulletMessage` | `string` | `'Go to slide {{index}}'` | Bullet label |
| `notificationClass` | `string` | `'v-surfer-notification'` | Class for the live region |
| `wrapperLiveRegion` | `boolean` | `true` | Put the live region on the wrapper |
| `id` | `string \| null` | `null` | Explicit id base |
| `scrollOnFocus` | `boolean` | `true` | Slide to a slide when it receives focus |
| `clickable` | `boolean` | `true` | Mark slides as clickable to AT |
| `autoplay` | `boolean` | `false` | Announce autoplay state |

## Component props

Same names and defaults as the config table above (each is an `<A11y>` prop).

## Slots

No slots.

## Host API

No `host.modules.a11y` API surface — it operates purely through DOM attributes.

## Events

No module-specific events.

## Markup

No visible chrome. The module sets ARIA attributes on the container, wrapper, and slides, and injects a visually-hidden live region:

```html
<span class="v-surfer-notification" aria-live="assertive" aria-atomic="true"></span>
```

## CSS classes

`v-surfer-notification` (configurable via `notificationClass`) — the visually-hidden live region. Styled by `v-surfer/css/a11y`.

## CSS variables

None.

## Example

```vue
<script setup>
import { Surfer, Item, Navigation, NavigationModule, A11y, A11yModule } from 'v-surfer'
import 'v-surfer/css'
import 'v-surfer/css/navigation'
import 'v-surfer/css/a11y'
</script>

<template>
  <Surfer
    :modules="[NavigationModule, A11yModule]"
    :slides-per-view="1"
    :config="{ a11y: { containerMessage: 'Featured products' } }"
  >
    <Item v-for="n in 5" :key="n">Slide {{ n }}</Item>
    <Navigation />
    <A11y />
  </Surfer>
</template>
```
